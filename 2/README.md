# Интерактивная карта русских говоров Удмуртии

Это самостоятельный Streamlit-прототип атласа, сделанный с другим интерфейсом, чем пример:
не одна карта с фильтрами, а исследовательская панель с разделами **Атлас**, **Карты**, **Пункты**, **Единицы**, **Таблица**, **ГИС**, **Инструкция**.

## Что реализовано по требованиям

- Территория: Удмуртия и прилегающие регионы в демо-данных.
- Интерактивная карта населённых пунктов с ландшафтом, диалектными единицами, комментариями.
- Фильтры по региону, району, карте-вопросу, единице, тексту.
- Ареалы и изоглоссы: автоматическая демонстрация по точкам выбранной карты.
- Поиск карт и паспорт карты.
- Выбор региона, района, населённого пункта и показ данных пункта.
- Поиск лингвистических единиц и показ их распределения.
- Таблица для удалённого редактирования через Google Sheets / CSV URL.
- Встроенная инструкция пользователя.
- Раздел по ГИС: назначение, функции, форматы, интерфейс и роль MapInfo, ArcGIS, QGIS, Maptitude, ГИС Аксиома, ГИС Панорама, NextGIS, ГИС ИНТЕГРО.

## Быстрый запуск

```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```


## Запуск именно через Streamlit

Основной входной файл для Streamlit Cloud — `streamlit_app.py`.
Локально приложение запускается так:

```bash
python -m pip install -r requirements.txt
python -m streamlit run streamlit_app.py
```

На Windows можно дважды открыть `start_streamlit.bat`, на macOS/Linux — выполнить `./start_streamlit.sh`.

Для Streamlit Community Cloud загрузите папку в GitHub и укажите:

- Repository: ваш репозиторий с проектом
- Branch: `main`
- Main file path: `streamlit_app.py`
- Python version: 3.12 или 3.11

## Удалённая таблица через Google Sheets

1. Создайте Google Sheets по шаблону `data/data_template.csv`.
2. Дайте редакторам доступ к таблице.
3. Опубликуйте лист как CSV или используйте ссылку вида:

```text
https://docs.google.com/spreadsheets/d/<SHEET_ID>/export?format=csv&gid=0
```

4. В Streamlit Cloud откройте **App -> Settings -> Secrets** и добавьте:

```toml
DATA_CSV_URL = "https://docs.google.com/spreadsheets/d/<SHEET_ID>/export?format=csv&gid=0"
REMOTE_TABLE_EDIT_URL = "https://docs.google.com/spreadsheets/d/<SHEET_ID>/edit"
BOUNDARY_GEOJSON_URL = ""
```

5. После правок в Google Sheets нажимайте в приложении **Обновить кэш данных**.

## Схема данных

Одна строка = один населённый пункт + один вопрос.

| Поле | Назначение |
|---|---|
| `region` | область / край / республика |
| `district` | район |
| `settlement` | населённый пункт |
| `settlement_type` | тип населённого пункта |
| `latitude`, `longitude` | координаты WGS84 |
| `landscape` | ландшафт / тип местности |
| `atlas_system` | ДАРЯ или ЛАРНГ |
| `question_type` | тип вопроса |
| `question_id` | код вопроса |
| `question` | формулировка вопроса |
| `linguistic_unit_1`, `linguistic_unit_2`... | варианты ответа / лингвистические единицы |
| `comment` | комментарий |
| `source` | источник записи |
| `year` | год записи |

Можно использовать русские заголовки: `регион`, `район`, `населенный пункт`, `вопрос`, `лингвистическая единица 1` и т. д. Приложение приведёт их к внутренней схеме.

## Типы вопросов ДАРЯ и ЛАРНГ

В демо и в валидаторе заложены такие значения:

- `ДАРЯ: фонетика`
- `ДАРЯ: морфология`
- `ДАРЯ: синтаксис`
- `ДАРЯ: лексика`
- `ЛАРНГ: лексика / природа`
- `ЛАРНГ: лексика / растительный мир`
- `ЛАРНГ: лексика / животный мир`
- `ЛАРНГ: лексика / человек`
- `ЛАРНГ: лексика / материальная культура`
- `ЛАРНГ: лексика / духовная культура`
- `ЛАРНГ: лексика / социальная сфера`

## Границы районов и ландшафты

Приложение поддерживает дополнительный GeoJSON-слой. Подготовьте его в QGIS/ArcGIS/NextGIS, проверьте CRS EPSG:4326 и вставьте HTTPS-ссылку в поле **GeoJSON границ/районов**.

## Важное замечание о демо-данных

`data/sample_dialects.csv` — учебный набор для проверки интерфейса. Его нужно заменить реальными полевыми материалами.
Автоматические изоглоссы строятся по выпуклой оболочке точек; для научного атласа лучше использовать проверенные слои GeoJSON.
